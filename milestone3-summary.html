Project Overview

Data story: The world's power infrastructure doesn't just produce electricity — it reshapes forests. Deforest Gump visualizes the spatial and statistical link between global power plant fuel types and deforestation pressure across 148 countries, enriched with population density context.

The visualization combines three open datasets into four linked, interactive views: an interactive Leaflet map, two D3 bar charts, a capacity pie chart, and a country-level correlation scatter plot. Every view responds to the same set of filters (fuel type, country, map viewport), creating a coherent exploratory experience.

feat: generate sampled deforestation GeoJSON from CSV
73193cc

The deforestation overlay had been wired up in the UI since Milestone 2 but the GeoJSON data file it tried to fetch did not exist, causing the toggle to silently fail. This commit creates the full preprocessing pipeline for the web layer.

The Python script reads the full 8-deforestation.csv (1.16 M rows), discards pixels with no loss (value ≤ 0), and samples every 20th remaining pixel to produce a browser-friendly file. Each feature stores the numeric driver (1–5) and the human-readable cause string. Targeting ~25 000 features keeps the file under 4 MB for fast initial load.

feat: implement population density overlay with log color scale
5fd9df5

Population density values span six orders of magnitude (near-zero to 5.1 M people per 10 km²), making a linear color scale useless. The preprocessing script applies a log-normalization: each feature stores both the raw population count and a norm value in [0, 1] computed as log(pop+1) / log(max+1).

The browser maps this to d3.interpolateYlOrRd, producing a yellow-to-red sequential scale that clearly separates sparse rural areas from dense urban centers. The population toggle now follows the same show/hide/legend pattern as the deforestation toggle. The buildPopulationToggle function was updated to accept the map argument it was previously missing.

feat: add country-level fossil fuel vs deforestation correlation scatter chart
1ddb9d6

This is the core visualization of Milestone 3 — the chart that makes the data story explicit. A Python script joins the two datasets on the ISO 3-letter country code: for each country it computes deforestation pixel count and fossil fuel share of total installed capacity (Coal + Gas + Oil + Petcoke). The join produces data for 148 countries.

The CorrelationScatter D3 class renders each country as a circle on a log-scale X axis (deforestation intensity) vs linear Y axis (fossil fuel %). Circle radius encodes plant count; fill color encodes the dominant fuel type using the existing FUEL_COLORS palette. Clicking a dot sets the country dropdown and fires the global filter, zooming all other charts and the map to that country. The placeholder card was removed.

When the deforestation overlay is enabled, the sidebar Plant Details card gains a secondary section: a breakdown of the five deforestation drivers visible within the current map bounds. All chart updates were previously instant, making it hard to follow what changed when panning the map or toggling a filter. This commit adds coordinated animations across all four D3 components using a shared 350 ms duration constant.

Bar charts: bars transition their y and height with easeCubicOut; Y-axis grid lines and tick labels slide to their new positions; hover brightens the bar to full opacity. A shared _updateBars() helper was extracted to the BarChart base class, removing duplication between the two subclasses. Pie chart: slices use attrTween("d", …) with d3.interpolate to morph arcs smoothly; hover brightens the hovered slice. Scatter chart: dots appear with a staggered fade-in (4 ms × index delay) from radius 0 to final size; hover transitions stroke and opacity.

Technical Architecture

Data pipeline

  • Raw GeoTIFFs → Python (rasterio) → CSVs with lat/lon/value/country
  • CSVs → three Python scripts → GeoJSON + JSON for the browser
  • Sampling: 1-in-20 deforestation, 1-in-30 population to keep files under 4 MB
  • Country join: ISO 3-letter code links power plants ↔ deforestation

Frontend modules

  • app.js — entry point, parallel data fetch, filter wiring
  • charts.js — D3 bar, pie, scatter chart classes
  • map.js — Leaflet marker creation and filter application
  • overlays.js — deforestation + population layer logic and stats export
  • ui.js — legend, chips, intro banner, sidebar stats renderer
  • constants.js — fuel/driver color maps and helper functions

Interactivity

  • All charts linked to map viewport via map moveend event
  • Fuel chips + country dropdown update markers and all 4 charts simultaneously
  • Scatter dot click sets country filter and re-fires filter change
  • Deforestation sidebar stats update on every pan/zoom

Libraries used

  • D3.js v5 — scales, axes, transitions, arc tweening, CSV/JSON loading
  • Leaflet.js — tiled base map, canvas renderer, circle markers, GeoJSON layers
  • TopoJSON — world country boundaries (vendored)
  • Google Fonts Inter — typography
